## Perché DEFERRED

Printing Press riduce token quando **agent MCP-based** (Claude Code, Cursor, Copilot) orchestra workflow via tool-calls. Il nostro stack attuale = **Python diretto via requests**, no MCP wrapping. Sostituire `requests.post(groq_url)` con `subprocess.run(['groq', 'chat', ...])` aggiunge dipendenza CLI + JSON serialization overhead = peggioramento, non miglioramento.

Vincolo #5 (zero-cost): adottare ora = supply chain risk + manutenzione 50+ binaries non usati = work creep senza ROI.

## Trigger conditions per RIVALUTARE (almeno 1 di 3)

### Trigger 1: ARGOS evolution → MCP agent orchestration

**Quando**: implementiamo orchestrazione workflow ARGOS via Claude Agent SDK + MCP, con agent autonomo che fa **50+ tool-calls/sessione** (es. agente gestisce automaticamente deal lifecycle senza founder HITL su ogni step).

**Come misurare**: contatore `cc-violations.jsonl` o equivalent log mostra `tool_call_count_per_session > 50` consistente.

**Stato attuale**: HITL strict D-07 forza founder approval su ogni step → niente autonomous agent. Probabile trigger S250+ post-revenue + payment evidence.

### Trigger 2: Token cost LLM ARGOS supera soglia €15/mese

**Quando**: stato `~/venture-os/state/costs.jsonl` mostra ARGOS LLM cost mensile > €15.

**Stato attuale**: €0/mese (Groq free tier 30 RPM / 1000 RPD copre fino ~10 deal attivi/giorno).

**Soglia raggiungimento**: quando ARGOS scala a 50+ deal/mese saturando free tier. Probabile S200+ con revenue evidence accumulata.
